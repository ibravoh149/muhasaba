# i18n Usage Guide

## Setup recap

The i18n system is built on three layers:
- **`i18next`** — the translation engine
- **`LanguageProvider`** — handles device detection, persistence, and RTL
- **`useFormatter`** — date, time, number, and currency formatting per locale

Both providers live in the root layout, so they're available everywhere in the app.

---

## Translating text

```tsx
import { useTranslation } from 'react-i18next';

export default function LoginScreen() {
  const { t } = useTranslation();

  return (
    <View>
      <Text>{t('auth.login')}</Text>
      <Text>{t('auth.email')}</Text>
      <Text>{t('auth.forgotPassword')}</Text>
    </View>
  );
}
```

### With variables

Add a placeholder in your locale file:

```ts
// src/i18n/locales/en.ts
welcome: 'Welcome, {{name}}!'
```

Then use it:

```tsx
t('common.welcome', { name: 'Ibrahim' })
// → "Welcome, Ibrahim!"
```

### Plurals

```ts
// src/i18n/locales/en.ts
streak: '{{count}} day streak',
streak_other: '{{count}} day streak',
streak_one: '1 day streak',
```

```tsx
t('common.streak', { count: 5 })  // "5 day streak"
t('common.streak', { count: 1 })  // "1 day streak"
```

## Changing language

Use `useLanguage` — for example on your onboarding language picker:

```tsx
import { LANGUAGE_LABELS, SUPPORTED_LANGUAGES, useLanguage } from '@/context/language';

export default function LanguagePicker() {
  const { language, setLanguage } = useLanguage();

  return (
    <View>
      {SUPPORTED_LANGUAGES.map((code) => (
        <Pressable key={code} onPress={() => setLanguage(code)}>
          <Text>{LANGUAGE_LABELS[code]}</Text>
          {language === code && <Text>✓</Text>}
        </Pressable>
      ))}
    </View>
  );
}
```

> Switching to `ar` automatically triggers an RTL reload. All other languages reload the UI without restarting.

## Formatting dates and numbers

```tsx
import { useFormatter } from '@/hooks/use-formatter';

export default function SomeScreen() {
  const fmt = useFormatter();
  const now = new Date();

  fmt.date(now)                  // "June 27, 2026"        → "٢٧ يونيو ٢٠٢٦" in Arabic
  fmt.time(now)                  // "2:30 PM"              → "٢:٣٠ م" in Arabic
  fmt.dateTime(now)              // "Jun 27, 2026, 2:30 PM"
  fmt.hijriDate(now)             // "2 Muharram 1448 AH"   → "٢ محرم ١٤٤٨ هـ" in Arabic
  fmt.relativeTime(now)          // "just now" / "now"
  fmt.number(1500000)            // "1,500,000"            → "١٬٥٠٠٬٠٠٠" in Arabic
  fmt.currency(99.99, 'USD')     // "$99.99"               → "٩٩٫٩٩ US$" in Arabic
  fmt.currency(99.99, 'EUR')     // "€99.99"               → "٩٩٫٩٩ €" in Arabic
}
```

### Relative time examples

```tsx
fmt.relativeTime(new Date(Date.now() - 30_000))      // "30 seconds ago"
fmt.relativeTime(new Date(Date.now() - 3_600_000))   // "1 hour ago"
fmt.relativeTime(new Date(Date.now() + 86_400_000))  // "tomorrow"
fmt.relativeTime(new Date(Date.now() - 7_776_000_000)) // "3 months ago"
```

## Showing both Hijri and Gregorian dates

A common pattern in Islamic apps — show both calendars together:

```tsx
const fmt = useFormatter();
const { t } = useTranslation();
const today = new Date();

<Text>{t('calendar.gregorian')}: {fmt.date(today)}</Text>
<Text>{t('calendar.hijri')}: {fmt.hijriDate(today)}</Text>
```

Output in English:
```
Gregorian: June 27, 2026
Hijri: 2 Muharram 1448 AH
```

Output in Arabic:
```
ميلادي: ٢٧ يونيو ٢٠٢٦
هجري: ٢ محرم ١٤٤٨ هـ
```

## Date manipulation with date-fns

For anything beyond formatting — arithmetic, ranges, comparisons — use `date-fns`:

```tsx
import {
  addDays,
  differenceInDays,
  endOfMonth,
  format,
  isSameDay,
  startOfWeek,
  subDays,
} from 'date-fns';

const today = new Date();

addDays(today, 7)                        // a week from now
subDays(today, 1)                        // yesterday
differenceInDays(new Date('2026-12-31'), today)  // days until end of year
isSameDay(today, new Date())             // true
startOfWeek(today, { weekStartsOn: 6 }) // Saturday (Islamic week start)
endOfMonth(today)                        // last day of current month
```

> Always use `date-fns` for manipulation and `useFormatter` for display. Never use `date-fns`'s own format functions for display — they don't respect the user's locale.

## Adding new translation keys

1. Add the key to `en`, `ar`, `de`, `fr` locale files
2. Do **not** update `ha`, `yo`, `ig` yet — those come later
3. TypeScript will infer the key shape from `en.ts`, so you'll get autocomplete

```ts
// src/i18n/locales/en.ts
export default {
  // ... existing keys
  myFeature: {
    title: 'My Feature',
    description: 'Does something useful',
  },
} as const;
```

Then use it:
```tsx
t('myFeature.title')
t('myFeature.description')
```
